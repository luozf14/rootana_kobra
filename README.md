# ROOTANA-Based Code for KoBRA Data Analysis

**Author:** Jongwon Hwang (Institute for Basic Science, Rare Isotope Science Project)

**Email:** <jwhwang@ibs.re.kr>

**Last updated:** December 24, 2025

## 1. Overview

This project is a **ROOTANA-based analysis codebase** for experimental data from KoBRA (Korea Broad acceptance Recoil spectrometer Array) at the RAON heavy-ion accelerator facility in Korea.

- **Offline analysis**: MIDAS raw data (`.mid`, `.mid.lz4`) -> `rakobra` -> `output000XXXXX.root`
- **Online monitoring**: `rakobra_server` launches a JSROOT (`THttpServer`) web monitoring server
- **Post-processing / physics analysis**: ROOT files are analyzed with the `KOBRA` class in `macros/kobra.cxx` and `macros/kobra.hxx`

## 2. Main Components and Features

This package consists of the following primary components.

### 2.1. `rakobra`

- **Role**: Reads KoBRA raw experimental data (`.mid`, `.mid.lz4`) and produces ROOT files for analysis.
- **Output**: Creates files named `output000XXXXX.root` in the current working directory. Helper scripts then move or organize those files.

### 2.2. `rakobra_server`

- **Role**: Online data analysis and monitoring server.
- **Main features**: Opens a web server with `THttpServer("http:9091")` and uses `online.htm` as the default page.
  - Default address: `http://localhost:9091`
  - Adds `MIDASSYS/resources` as a web-accessible location

### 2.3. `KOBRA` Class (`macros/kobra.cxx`, `macros/kobra.hxx`)

- **Role**: C++ class for analyzing ROOT files generated by `rakobra`, typically used from ROOT macros.
- **Main features**:
  - Loads and manages run data for multiple experiments and isotopes such as 18O, 19O, and 20O
  - Draws particle-identification (PID) diagrams such as DeltaE-E and A/Q-Z
  - Defines and applies particle gates with `TCutG`
  - Produces and fits histograms for momentum distributions, angular distributions, and related observables
  - Accesses and uses scaler data through `TScalerData`
  - Computes PPAC efficiency, live time, and related quantities
  - Provides helper functions for cross-section analysis and momentum-distribution analysis

### 2.4. `scripts/scaler.py`

- **Role**: Loads KoBRA scaler CSV files (`misc/scaler_ext_{isotope}.csv`) and prints **run summaries** such as totals and average rates. It can also draw plots when needed.

## 3. Requirements

- **CMake**: 3.10+
- **C++ compiler**: `clang++` or `g++`
- **ROOT**: Must be discoverable through `find_package(ROOT ...)` in CMake
- **MIDAS + ROOTANA**: Header and library paths must be supplied through environment variables
  - `MIDASSYS`: MIDAS installation root
  - `ROOTANASYS`: ROOTANA installation root, or the ROOTANA build/install directory
  - `ROOTSYS`: ROOT installation root. This projectâ€™s `CMakeLists.txt` references `ROOTSYS/etc/cmake`
- **Python 3.x (optional)**: Required when using scripts under `scripts/`
  - `pandas`, `matplotlib`, `numpy`

## 4. Installation

Below is a reproducible clean setup/build procedure, using macOS as the baseline example.

### 4.1. Install ROOT and Set Up the Environment

- **Option A (recommended: package manager)**: Install ROOT with Homebrew or conda-forge
- **Option B**: Build ROOT from source or install the official binary distribution

In either case, the following should work in your terminal:

```bash
root-config --version
root -l -b -q
```

This project also references `ROOTSYS` in `CMakeLists.txt`, so you need one of the following:

- **Recommended**: Source the `thisroot.sh` script provided by your ROOT installation
- **Or**: Export `ROOTSYS` manually

### 4.2. Install MIDAS / ROOTANA and Set Up the Environment

This code links against the ROOTANA (`rootana`) and MIDAS (`midas`) libraries. Before building, the following environment variables must be defined:

```bash
export MIDASSYS=/path/to/midas
export ROOTANASYS=/path/to/rootana
export ROOTSYS=/path/to/root
```

Notes:

- On macOS, you may need `DYLD_LIBRARY_PATH` if dynamic-library loading fails.
- If you built ROOTANA or MIDAS manually, verify that the required libraries exist under `ROOTANASYS/lib` and `MIDASSYS/lib`.

### 4.3. (Optional) Install Python Dependencies

```bash
python3 -m pip install --upgrade pip
python3 -m pip install pandas matplotlib numpy
```

### 4.4. Build This Project with CMake

From the repository root:

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j
```

Build artifacts by default:

- Executables: `build/rakobra`, `build/rakobra_server`, `build/midcheck`
- Libraries: `build/libkobra.*` and generated ROOT dictionary files

**Important**: `ana.sh` and `batch.sh` call `./rakobra` by default. Choose one of the following:

- **Option A (recommended)**: Run the binaries directly from `build/`
- **Option B**: Create symbolic links in the repository root for script compatibility

```bash
ln -sf build/rakobra ./rakobra
ln -sf build/rakobra_server ./rakobra_server
ln -sf build/midcheck ./midcheck
```

## 5. Usage

### 5.1. `rakobra`: Generate ROOT Files

```bash
# Offline analysis input: .mid or .mid.lz4
./rakobra /path/to/run00042.mid.lz4
```

In practice, offline processing is usually run in batches with `ana.sh`, which processes a run range and organizes output files, for example into `root/phys` or `root/comm`.

```bash
# Usage:
#   ./ana.sh [comm|phys] (start_run) (end_run)
./ana.sh phys 251 254
./ana.sh comm 1 10
```

`ana.sh` assumes the following default input paths:

- `phys`: `./rawdata/KO_24_30/run{RUN}.mid.lz4`
- `comm`: `./rawdata/KO_comm/run{RUN}.mid.lz4`

It then moves the generated `output000{RUN}.root` files into `./root/phys` or `./root/comm`, respectively.

### 5.2. `KOBRA` Class: Analyze ROOT Files

Run a macro that uses the `KOBRA` class from the ROOT prompt.

```cpp
// Example: my_analysis.C
void my_analysis() {
    // Load the KOBRA class implementation if needed
    // gSystem->Load("libKOBRA.so"); // if built as a shared library
    // gROOT->ProcessLine(".L macros/kobra.cxx+"); // JIT compile

    KOBRA k(KOBRA::Expt::Phys, 251); // example: load run 251 from the 18O experiment
    // Or load multiple runs:
    // std::vector<int> runs = KOBRA::o19; // all 19O runs
    // KOBRA k(KOBRA::Expt::Phys, runs);

    k.SetAlias(); // define aliases for physics quantities
    k.AddCuts("cut"); // directory containing gate files, for example ./cut

    // Draw PID diagram (0: default, 1: show counts, 2: show ratios)
    k.DrawPIDC(1);

    // Draw the momentum distribution for a specific isotope, for example "o19"
    // k.DrawMomDist("o19");

    // Save the result
    // gPad->GetCanvas()->SaveAs("pid_o18_run251.png");
}
```

Run ROOT with:

```bash
root -l my_analysis.C
```

### 5.3. `scripts/scaler.py`: Analyze Scaler Data

```bash
python3 scripts/scaler.py <isotope_name>
# Example: for 18O data, assuming misc/scaler_ext_18o.csv exists
python3 scripts/scaler.py 18o
```

The script reads `misc/scaler_ext_{isotope}.csv` and prints a `Run Summary`. If you uncomment the relevant lines in the script, it can also draw `ppac_rate` plots for selected runs and overlay `aoq_reft.csv`.

### 5.4. `rakobra_server`: Online Monitoring

```bash
./rakobra_server
```

After the server starts, open `http://localhost:9091` in a browser. The default page is `online.htm`.

### 5.5. `scripts/plot_lebt_beamdump.py`: Plot LEBT and BeamDump Together

This script overlays `scripts/beam_data.py` (LEBT) and `scripts/control_data.py` (BeamDump) on the **same time axis**, using left and right y-axes.

```bash
# Recommended to run from the repository root because the script uses relative misc/*.pkl paths
python scripts/plot_lebt_beamdump.py \
  --date-start 2024-08-13 --time-start 13:05:00 --time-end 13:50:00 \
  --beamdump-col 9 \
  --out misc/lebt_beamdump_2024-08-13.png
```

If the time bases are misaligned, you can apply a **timestamp offset**:

```bash
# Shift BeamDump by +2 seconds to align it with LEBT
python scripts/plot_lebt_beamdump.py \
  --date-start 2024-08-13 --time-start 13:05:00 --time-end 13:50:00 \
  --offset-beamdump +2s
```

If you want the script to determine the offset automatically, use `--auto-offset`. It scans a range and finds the delta t that maximizes the Pearson correlation.

```bash
python scripts/plot_lebt_beamdump.py \
  --date-start 2024-08-13 --date-end 2024-08-15 \
  --time-start 13:05:00 --time-end 13:50:00 \
  --auto-offset --auto-offset-target beamdump \
  --auto-offset-range='-5s,5s' --auto-offset-step 100ms \
  --auto-offset-only
```

If you want to inspect multiple dates, specify either `--date-end` for a date range or `--dates` for an explicit list. The script automatically splits the output into per-date subplots in a single figure.

```bash
# Date range saved as one figure with subplots
python scripts/plot_lebt_beamdump.py \
  --date-start 2024-08-13 --date-end 2024-08-15 \
  --time-start 13:05:00 --time-end 13:50:00 \
  --ncols 3 \
  --out misc/lebt_beamdump_2024-08-13_to_2024-08-15.png

# Explicit comma-separated date list
python scripts/plot_lebt_beamdump.py \
  --dates 2024-08-13,2024-08-15,2024-08-18 \
  --time-start 13:05:00 --time-end 13:50:00 \
  --ncols 3 \
  --out misc/lebt_beamdump_multi.png
```

If the BeamDump data is not in column `9`, first inspect the available columns and then pass either a numeric index or a column name to `--beamdump-col`.

```bash
python scripts/plot_lebt_beamdump.py --date-start 2024-08-13 --time-start 13:05:00 --time-end 13:50:00 --list-control-columns
```

To draw a **2D LEBT-BeamDump correlation plot**, use `--correlation`. Internally, the script merges the two time series with `merge_asof`, and `--tolerance` controls the maximum allowed time difference.

```bash
python scripts/plot_lebt_beamdump.py \
  --date-start 2024-08-13 --date-end 2024-08-15 \
  --time-start 13:05:00 --time-end 13:50:00 \
  --beamdump-col 9 \
  --offset-lebt 0s --offset-beamdump +2s \
  --correlation --corr-kind hexbin --tolerance 2s \
  --save-merged-csv misc/lebt_beamdump_merged.csv \
  --out misc/lebt_beamdump_corr.png
```

## 6. Data Files

- **KoBRA raw data**: `.mid` format (MIDAS standard)
- **ROOT files for analysis**: Generated by `rakobra`, typically named `output000XXXXX.root`. These may include event data in `TTree` form as well as objects such as `TScalerData`.
- **Scaler data**: `misc/scaler_ext_{isotope}.csv`, with columns such as `run, ts, ppac, f2pl, f3pl, hwtriga, hwtrigb, refp, reft`
- **A/Q vs reft data**: `misc/aoq_reft.csv` as an example input
- **PID gate files**: Text files under the `cut/` directory. Each file defines A/Q and Z coordinates for a specific isotope gate.

## 7. Parameter Files (`pars/` Directory)

The `pars/` directory contains calibration parameters for each detector in text-file format. These files are loaded into detector analysis classes through `analist.txt` and are used to convert raw data into physical quantities.

### 7.1. PPAC Parameter Files

**Files:**
- `PAR_F2UPPAC.txt`, `PAR_F2DPPAC.txt`, `PAR_F3UPPAC.txt`, `PAR_F3DPPAC.txt`, `PAR_SCPPAC.txt`

**Format (5 lines):**
```text
Line 0: chs          Channel numbers (space-separated)
Line 1: offset       Per-channel offset values (space-separated)
Line 2: factor       Per-channel calibration factors (space-separated)
Line 3: tdc_cut      Lower and upper TDC bounds (2 values)
Line 4: tsum_cut     X lower X upper Y lower Y upper (4 values)
```

**Description:**
- `chs`: TDC channel numbers used by the detector
- `offset`: Time-offset correction for each channel in ns
- `factor`: Time calibration factor for each channel in ns/ch
- `tdc_cut`: Valid TDC range `[lower, upper]`
- `tsum_cut`: Valid tsum ranges for X and Y `[x_lower, x_upper, y_lower, y_upper]`

**Used by:** `TPPACAna`

### 7.2. LPPAC (Large PPAC) Parameter Files

**Files:**
- `PAR_F1UPPAC.txt`, `PAR_F1DPPAC.txt`

**Format (6 lines):**
```text
Line 0: chs           Channel numbers (space-separated)
Line 1: offset        Per-channel offset values (space-separated)
Line 2: factor        Per-channel calibration factors (space-separated)
Line 3: tdc_cut       Lower and upper TDC bounds (2 values)
Line 4: tsum_cut      X lower X upper Y lower Y upper (4 values)
Line 5: txsum_offset  X tsum offsets [left, center, right] (3 values)
```

**Description:**
- Same structure as PPAC, but with an additional `txsum_offset` parameter
- `txsum_offset`: Offset corrections for the left, center, and right X tsum regions

**Used by:** `TLPPACAna`

### 7.3. LPPACX (Large PPAC, X Only) Parameter Files

**Files:**
- `PAR_F1UPPACX.txt`

**Format (6 lines):**
```text
Line 0: chs           Channel numbers (space-separated)
Line 1: offset        Per-channel offset values (space-separated)
Line 2: factor        Per-channel calibration factors (space-separated)
Line 3: tdc_cut       Lower and upper TDC bounds (2 values)
Line 4: tsum_cut      X lower X upper (2 values)
Line 5: txsum_offset  X tsum offsets [left, center, right] (3 values)
```

**Description:**
- Similar to LPPAC, but only the X direction is measured, so `tsum_cut` contains two values instead of four

**Used by:** `TLPPACXAna`

### 7.4. Plastic Parameter Files

**Files:**
- `PAR_F2Plastic.txt`, `PAR_F3Plastic.txt`
- `PAR_F1PPT.txt`, `PAR_F2PPTU.txt`, `PAR_F2PPTD.txt`, `PAR_F3PPTU.txt`, `PAR_F3PPTD.txt`
- `PAR_F2SSDT.txt`, `PAR_F3SSDT.txt`

**Format (5 lines):**
```text
Line 0: chs         Channel numbers (space-separated)
Line 1: offset      Per-channel offset values (space-separated)
Line 2: factor      Per-channel calibration factors (space-separated)
Line 3: tdc_cut     Lower and upper TDC bounds (2 values)
Line 4: tdiff_cut   Lower and upper time-difference bounds (2 values)
```

**Description:**
- `chs`: TDC channel numbers used by the plastic detector
- `offset`: Time-offset correction for each channel
- `factor`: Time calibration factor for each channel
- `tdc_cut`: Valid TDC range
- `tdiff_cut`: Valid time-difference range between the left and right channels

**Used by:** `TPlasticAna`

### 7.5. DPlastic (Digitizer Plastic) Parameter Files

**Files:**
- `PAR_F2DPlastic.txt`, `PAR_F3DPlastic.txt`

**Format (7 lines):**
```text
Line 0: chs         Channel numbers (space-separated)
Line 1: t_offset    Time-offset values (space-separated)
Line 2: t_factor    Time calibration factors (space-separated)
Line 3: a_offset    Energy/ADC offset values (space-separated)
Line 4: a_factor    Energy/ADC calibration factors (space-separated)
Line 5: tdc_cut     Lower and upper TDC bounds (2 values)
Line 6: adc_cut     Lower and upper ADC bounds (2 values)
```

**Description:**
- `chs`: Digitizer channels used by the DPlastic detector
- `t_offset`: Time-offset correction for each channel
- `t_factor`: Time calibration factor for each channel
- `a_offset`: Energy/ADC offset correction for each channel
- `a_factor`: Energy/ADC calibration factor for each channel
- `tdc_cut`: Valid TDC range
- `adc_cut`: Valid ADC range

**Used by:** `TDPlasticAna`

### 7.6. SSD (Silicon Strip Detector) Parameter Files

**Files:**
- `PAR_F2SSD.txt`, `PAR_F3SSD.txt`

**Format (5 lines):**
```text
Line 0: chs       Channel numbers (space-separated)
Line 1: ped       Per-channel pedestal values (space-separated)
Line 2: offset    Per-channel offset values (space-separated)
Line 3: factor    Per-channel calibration factors (space-separated)
Line 4: adc_cut   Lower and upper ADC bounds (2 values)
```

**Description:**
- `chs`: ADC channel numbers used by the SSD
- `ped`: Pedestal values for baseline correction
- `offset`: Energy-offset correction for each channel
- `factor`: Energy calibration factor for each channel
- `adc_cut`: Valid ADC range

**Used by:** `TSSDAna`

### 7.7. How Parameter Files Are Used

Parameter files are mapped to detectors in `analist.txt`:

```text
detector_name  data_type  bank_name  analysis_type  parameter_file_path  [num_channels]
```

Examples:

```text
f2ssd    KoBRAADC  F3SI    SSD       pars/PAR_F2SSD.txt       16
f3dpla   KoBRADIG  V1730   DPlastic  pars/PAR_F3DPlastic.txt
```

Parameter files support comments starting with `#`, and blank lines are ignored.
